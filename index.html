KeyBase

A fast, form-driven desktop app for managing key-duplication shop records.

Overview

KeyBase is a lightweight desktop application (Java + Swing) that focuses on fast data entry, image capture, printing, search and Excel export for a key-duplication / locksmith workflow. It uses an embedded H2 database and provides a compact, keyboard-driven UI for shop workflows.

Features

  • Form-driven main window: add records with fields for Name, Phone (10-digit validated), Vehicle No, ID No, Key No/Model, Key Type, Purpose, Remarks, Quantity, Amount and Date.
  • Keyboard shortcuts: Ctrl+S (save), Ctrl+F (search), Ctrl+E (export to Excel), Ctrl+P (print today's records).
  • Service & payment markers stored in Remarks (Duplicate/In-shop/On-site and Cash/UPI), kept machine-parsable without schema changes.
  • Integrated webcam capture: preview, Capture, Delete; temporary cache until record saved; preference to disable camera features.
  • Records table view: non-editable rows, selection, right-click menu (View Details, Print, Export CSV for a record).
  • Printing: formatted monospaced report for today with summary (count, quantity, total amount).
  • Excel export (XLSX) with configurable columns and typed/widthed cells.
  • Search, Preferences (including camera control), Help/Readme and About dialogs.

How the site is published

This repository includes a small GitHub Pages site in the docs/ directory. To publish the site automatically you can push these changes to the main branch — a workflow in .github/workflows/pages.yml is provided which publishes docs/ to GitHub Pages.

After the workflow runs, the site will be available at:

https://SamSunny4.github.io/KeyBase-Releases/

Local preview

To preview locally, serve the docs/ directory with a simple HTTP server. For example (requires Python):

python -m http.server 8000 --directory docs

Then open http://localhost:8000/.

Notes for maintainers

  • The GitHub Actions workflow will publish docs/ on every push to main. You can disable or change this behavior in .github/workflows/pages.yml.
  • If you prefer, instead of using the workflow you can enable Pages in the repo settings and set the source to main /docs.
